# Программирование полёта

**English:** [FLIGHT_PROGRAM_EN.md](FLIGHT_PROGRAM_EN.md)

---

В меню паузы (**Esc** / **Пробел**) выберите **«Программирование полёта»**. Откроется редактор: ограниченный **Python**, который вызывается **каждый шаг симуляции**, пока включён режим **Авто** и полёт не завершён.

## Обязательная точка входа

Нужно определить функцию:

```python
def tick(sim, ap):
    ...
```

- **`sim`** — только чтение: высота, скорости, флаги стадий, что разрешено сделать (`can_*`).
- **`ap`** — команды на этот кадр: сбросы, парашюты, двигатель, слайдер тяги (0…1).

Подсказки по именам — в правой панели редактора; **Tab** дополняет префикс после `sim.` или `ap.`.

## Что разрешено в языке

- Встроенные функции: `abs`, `min`, `max`, `round`, `len`, `range`, `enumerate`, `zip`, `sum`, `any`, `all`, `pow`, `divmod`, типы `int`, `float`, `bool`, `str`, `tuple`, `list`, `dict`, `set`.
- Модуль **`math`** (например `math.sqrt`, `math.sin`).
- Функция **`sleep(t)`** в глобальной области (см. ниже).
- **`import` и обращение к файлам недоступны** (песочница не полная; не запускайте ненужный код из недоверенных источников).

| Имя | Тип / смысл |
|-----|----------------|
| `altitude_m` | Высота над локальной поверхностью, м |
| `vertical_speed_mps` | Вертикальная скорость (+вверх), м/с |
| `horizontal_speed_mps` | Горизонтальная скорость, м/с |
| `time_s` | Модельное время с начала, с |
| `result` | Исход симуляции (`running` / …) |
| `engine_on` | Двигатель включён |
| `heatshield_jettisoned`, `drogue_deployed`, `main_deployed`, `chute_jettisoned` | Флаги стадий |
| `can_heatshield_jettison`, `can_drogue`, `can_main`, `can_chute_jettison` | Можно ли выполнить действие |
| `fuel_kg` | Топливо, кг |
| `g_load` | Перегрузка, g |
| `atm_pressure_bar`, `internal_temp_c`, `heatshield_skin_temp_c` | Давление, температура отсеков, температура обшивки (пока теплозащита не сброшена) |

| `stat(key, default)` | Доп. поля из HUD (`h_km`, `v_vert`, `dist_m`, …) |

## Объект `ap` (действия)

| Вызов | Действие |
|-------|-----------|
| `request_heatshield_jettison()` | Запрос сброса теплозащиты |
| `request_drogue()` | Тормозной парашют |
| `request_main()` | Основной парашют |
| `request_chute_jettison()` | Сброс парашюта |
| `set_engine(on)` | Вкл/выкл двигателя |
| `set_throttle_slider(v)` | Слайдер тяги 0…1 |
| `clamp(x, lo, hi)` | Ограничение числа |
| `sleep(t)` | То же, что глобальная `sleep(t)` (см. ниже) |

### `sleep(t)` — пауза по модельному времени

**`sleep(t)`** (в теле `tick` можно вызывать и как **`sleep(t)`**, и как **`ap.sleep(t)`**) приостанавливает автопилот на **`int(t)`** полных секунд **модельного времени** (`PhysicsModel.time_s`, то же, что **`sim.time_s`**). Дробная часть отбрасывается (как у встроенного **`int`**), **`t < 0`** даёт **`0`**:

- текущий вызов `tick` **сразу завершается** — код **после** `sleep` в этом же вызове **не выполняется**;
- следующие кадры **до** наступления момента `sim.time_s + int(t)` **полностью пропускают** `tick` (команды из программы не выдаются);
- после этого `tick` снова вызывается **с начала** (как обычно каждый шаг).

При проверке при **сохранении** `sleep` не оказывает эффекта. После **рестарта** миссии таймер сбрасывается.

Управление паузой **`sleep` не отменяет**: пока запланировано ожидание, вызовы `tick` пропускаются, но модельное время идёт как обычно (в т.ч. при ускорении времени).

## Редактор кода

Пока открыт редактор, **симуляция на паузе**. Ссылка на этот файл отображается в правой панели (**См. FLIGHT_PROGRAM_RU.md** / в английском интерфейсе — **FLIGHT_PROGRAM_EN.md**).

### Кнопки

| Кнопка | Действие |
|--------|-----------|
| **Сохранить** | Компиляция, проверка `tick` (см. ниже), при успехе — закрытие редактора и возврат в меню паузы |
| **Отменить** | Восстановить текст последнего **успешно сохранённого** варианта, закрыть редактор (**Esc** делает то же) |

### Подсказки и ввод

- **Правая панель** — списки членов **`sim`** и **`ap`**; **клик** по строке вставляет её в позицию каретки. Прокрутка — **колёсико** над панелью.
- **Подсветка синтаксиса** — ключевые слова Python, строки, комментарии, числа, имена **`sim`** / **`ap`**.
- **Tab** — автодополнение, если перед курсором префикс вида **`sim.`** или **`ap.`**; иначе вставляются **4 пробела** (как отступ).

### Выделение и буфер обмена

- **Мышь**: перетаскивание по тексту задаёт выделение; **Shift+клик** расширяет выделение от прежней каретки.
- **Shift+стрелки**, **Shift+Home/End** — расширить или сузить выделение.
- **Ctrl+A** — выделить всё.
- **Ctrl+C** / **Ctrl+X** / **Ctrl+V** — копировать, вырезать, вставить (многострочный текст поддерживается). Системный буфер используется через Pygame, при недоступности — **внутренний** буфер только на время сессии.
- **Backspace** и **Delete** сначала удаляют выделение, если оно есть. При **удержании** клавиш срабатывает **повтор** (как в обычных полях ввода).

### Прочее

- **Ctrl+S** — то же, что **Сохранить**.
- Редактирование с зажатым **Ctrl** не порождает лишних символов из раскладки (шорткаты обрабатываются отдельно).

## Сохранение и проверка

**Сохранить** выполняет по порядку:

1. **Компиляция** ограниченного Python; при синтаксической ошибке или отсутствии `tick(sim, ap)` активная сохранённая программа **не меняется**, в редакторе показывается ошибка.
2. **Один пробный вызов** `tick` с «пустым» `ap` (без команд к модели и без реального `sleep`). При исключении сохранение **отменяется**, редактор остаётся открытым.
3. При успехе новый текст становится программой **Авто**, редактор закрывается.

## Ошибки во время полёта

Исключения в **`tick`** (кроме внутренней логики **`sleep`**) переводят режим **Авто** в выключенное состояние и показывают сообщение. Пользовательский код не должен перехватывать **`BaseException`** целиком — иначе можно сломать обработку **`sleep`**.
